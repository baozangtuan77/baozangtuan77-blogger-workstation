# 01 · 爆款选题分析

> 「博主AI工作台」能力。**先读工作台 `SKILL.md` 的核心公理 + 方法底座 + 禁用清单**，本文件只写这个能力特有的流程。
> 它帮博主：拆解 TA 赛道的爆款，提炼"为什么火"的、**能复用的选题方向**——不是抄标题，是抄底层的触发逻辑。
> **什么时候用**：你看到别人的爆款、想搞懂为什么火并复用时用本能力。只是想直接要一批自己的选题 → 用 `02 批量生成选题`。
> 例子用母婴 / 家居做示范，别的赛道照换。

## 第一步：先找到 10 条"真爆款"当样本（这步最关键，别偷懒）

**上哪找**（任选，凑够 10 条就行）：
1. **小红书搜索框** → 搜你赛道核心词 → 点「最热」排序。〔母婴示范〕搜"辅食""带娃神器""幼儿园准备"；〔家居示范〕搜"小户型收纳""出租屋改造""厨房好物"。
2. **你对标账号的主页** → 进 3-5 个对标账号，翻他们的置顶 / 高赞笔记。
3. **小红书「发现」页 / 你赛道的分类页** → 刷一刷，记下反复出现的爆款。
4. （进阶·可选）**第三方榜单**：蒲公英、灰豚、新红等能看赛道热门笔记，部分功能收费。

**怎么算"真爆款"**（挑的时候过这几关，别乱抓）：
- ✅ **近 1-2 个月**的（别翻两年前的，玩法过时了）
- ✅ **点赞过千**，或明显**远超那个账号自己的平均**水平
- ✅ 和**你赛道、你够得着的粉丝量级**接近——别全挑百万粉大 V，他们自带流量，规律你学不来
- ✅ 尽量覆盖**不同方向**（别 10 条全一个套路），方便后面提炼出多个角度

**每条抄下三样**：标题、点赞数、收藏数。（收藏尤其重要——**收藏比点赞高的是"工具型干货"**，玩法和情绪型完全不同。）

**两种拿法**：手动照上面找（免费）；或用 **TikHub API** 抓（⚠️ 收费、需你自己的 key，仅 Claude Code / Agent 里能用，豆包用不了）。

## 输入

```
【我的账号档案】（粘你填好的那份）
【我赛道近期 10 条爆款】（每条：标题 + 点赞 + 收藏）
1. ……
（到 10）
```

---

你按下面五步做，全程守 SKILL.md 的公理——尤其**"先判断再产出"**和**"给标尺不给枷锁"**（下面的分类和框架是帮你看清规律、判断方向，不是让你机械贴标签）。

### Step 1 · 先过质量门
看用户贴的够不够：**少于 5 条、或没有点赞/收藏数据 → 先说"再多贴几条、把赞藏补上，样本太小看不出规律"，别硬分析。** 够了再往下。

### Step 2 · 逐条拆（看触发逻辑，不是复述）
每条爆款一行给出：
- **戳的是 痛点 / 痒点 / 爽点 哪个**，怎么戳的（一句）
- **标题套路**（数字清单 / 反常识 / 痛点点名 / 身份共鸣 / 结果前置…），它好在哪
- **收藏 vs 点赞**：藏 > 赞 标「工具型干货」，赞 > 藏 标「情绪型」——这两类选题打法完全不同

### Step 3 · 提炼 3 个可复用方向，每个再裂变出几条新选题（重点）
不是抄标题，是抄"它为什么火"那个机制。每个方向给：
- 一句话**机制**（〔母婴示范〕"把一个喂养焦虑瞬间，拍成一步步的解法"｜〔家居示范〕"用改造前后的强对比放大爽感"）
- 标它主要服务哪层商业目的（扩漏斗 / 建信任 / 促转化）
- **用「爆款元素裂变法」从这一个爆点裂变出 2-3 条你赛道能落地的新选题**——别只憋出一条。
  方法：抓住这条爆款的核心话题，沿 **人群 / 价值点 / 时间 / 地点** 四维各换一刀（换个人群、换个具体需求、换个时间节点、换个场景），就是一条新选题。〔母婴示范〕"夏天坐月子"核心话题 → 换人群「剖腹产妈妈夏天坐月子」、换价值「夏天坐月子怎么变瘦」。四维里**别只在时间维度换**（618→双十一那种最偷懒），换人群往往更精准、更好转化。

### Step 4 · 给每条裂变选题过一遍「五维验证」，挑出真能做的
裂变出来的选题不是都值得拍。用「爆款选题五维验证法」快速过滤：让用户（或你在 Agent 模式）拿这条选题的关键词去小红书搜一下，看是否同时满足——① **互动过千**（赞+藏+评合计）② **半年内**还在出新爆 ③ **前三屏爆款密度高**（多人做爆才是刚需，一条是运气）④ **不是清一色品牌广告号**（没预算跑不出来）⑤ **不是清一色 10万+ 大博主**（头部自带流量池，素人复现不了）。
- 五维全过 → 标「✅ 已验证·优先做」；只过 3-4 维 → 标「⚠️ 谨慎」并说缺哪维；明显过不了（全是大 V / 全是广告 / 半年没新爆）→ 直接划掉，别凑数。
- **降级提示**：用户没法当场搜（豆包 / 没贴搜索结果），就把五个维度作为**自检清单**给 TA，让 TA 自己去搜了再定，别替 TA 拍"已验证"。

### Step 5 · 商业落点
留下来的方向 / 选题里，哪个更容易**接商单**、哪个更适合**涨粉 / 引流**，一句话点。

## 输出模板（按这个结构给）

```
【样本判断】10 条里 X 条工具型 / Y 条情绪型，整体规律：……

【逐条拆解】
1. [标题] | 痛点(怕选错) | 套路:反常识 | 藏>赞·工具型
2. ……

【3 个可复用方向 + 裂变选题】
方向一：[机制一句话] · 服务：建信任
   · 裂变选题1（换人群）：…… ｜ 五维验证：✅ 已验证·优先做
   · 裂变选题2（换时间）：…… ｜ 五维验证：⚠️ 谨慎（缺"密度"，前三屏只1条爆）
方向二 / 方向三：同上

【商业落点】留下来的里最该先做 [某条]，因为……（接商单 / 涨粉）
```

## 自检（发给用户前逐条过）
- [ ] 每个方向都说清了"为什么火"的**机制**，不是复述标题？
- [ ] 每个方向都**裂变出 ≥2 条**"我赛道能落地"的新选题（不是抄来的，且没只在时间维度换）？
- [ ] 每条裂变选题都过了**五维验证**、标了已验证 / 谨慎 / 划掉（不能当场搜的就把五维作清单交给用户，没替 TA 拍"已验证"）？
- [ ] 没编用户没贴的爆款 / 数据、没编"已验证"结论？
- [ ] 商业落点说清了服务哪层目的？

## 正反例（提炼方向时）
- ✅「藏 > 赞说明大家当工具收藏，方向是"做能被反复查的清单型"，比如《0-3 岁辅食红黑榜》」——讲清机制 + 可落地
- ❌「她标题用了数字，你也用数字」——只抄表面、没讲为什么、换个选题就不能用

## 禁用（叠加 SKILL.md 通用禁用）
- 别编用户没贴的账号 / 爆款；别给"爆款概率 ⭐"这种无依据分；别拿别赛道的爆款举例。
