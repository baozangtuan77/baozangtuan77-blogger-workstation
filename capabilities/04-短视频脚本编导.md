# 短视频脚本编导

> 「博主AI工作台」能力。**先读 `SKILL.md` 的公理 + 方法底座 + 禁用**，本文件只写本能力流程。
> 它帮博主：把一个选题 + 你的真实感受/素材，先**策划成「结构 + 切入点 + 大纲」**，停下来跟你对方向，再**编导成能直接拍的口播脚本 + 拍摄备注**。两段一气呵成，但中间一定停——别写到一半发现方向错了。
> 这一篇用的是七七拆 45 条爆文提炼出来的真实框架（脚本结构十三式 / 前5秒切入八公式 / 结尾三钩子 / 活人感五要素），不是套路化模板。
> **什么时候用**：选题定了、要写脚本。想要一批选题 → `批量生成选题`；稿子写好了要改 → `活人感改稿`；接的是商单 → `商单脚本`。
> 例子用母婴/家居做〔示范〕，别的赛道照换。

## 输入

```
【我的账号档案】（粘你填好的）
【这次的选题】：______
【内容类型】（不确定就空着，我帮你判断）：种草合集 / 单品商单 / 干货教学 / 选购科普 / 避坑揭秘 / 生活vlog / 创意信息差
【我的真实感受 / 素材】（3 条就够，越真越好）：
- ______（你身上真实发生的小事、踩过的坑、跟粉丝聊过的话）
- ______
- ______
【时长】：默认 60-90 秒
```
> 没素材？只填选题也行——进 Phase 1 我先帮你搭框架，到该有素材的地方标「（这里补你的真事）」。

## Phase 1 · 内容策划（先想清楚：定结构 → 找切入点 → 选开头 → 出大纲）

### ① 定结构（用「爆款脚本结构十三式」里最对路的那一式，不是我编的类型）

> 结构是骨架——同一个选题用不同结构拍，数据能差 5-10 倍。看选题 + 内容类型，从下面挑**最对路的一式**，一句话说为什么选它。**实在拿不准就默认「清单遍历」——七七拆 32 条爆款里近一半都在用它，最稳。**

| 结构 | 一句话适用 |
|---|---|
| **清单遍历**（式2）| 多个产品/方法逐一展示，每个独立成小故事；最稳的爆款结构，拿不准就用它 |
| **长期主义清单**（式8）| 同是清单，但用「不会闲置/花一次钱」当筛选器，筛出愿意买好货的人 |
| **痛点清单干货**（式4）| 选购/科普类：先列一堆让人头大的名词制造焦虑，再分类讲清；收藏通常高于点赞 |
| **教程跟练**（式5）| 方法教学：先甩结果再拆步骤，每步带编号方便暂停截图 |
| **结果前置 vlog**（式6）| 生活方式：标题就是结论，视频是「证明过程」，真实镜头比口播更服人 |
| **我家改造自白**（式9）| 空间/做法被质疑过：用「别人说我离谱，但这才对」的争议张力，观众投票站你这边 |
| **结论先行-逐条论证**（式10）| 避坑/纠偏：先抛反常识结论，再逐条证明「大家这样做错在哪、正确怎么做」 |
| **废物老公变惊喜**（式3）| 单品商单：用「质疑→真香」逐个展示卖点，看完像「我也被说服了」 |
| **闯祸→救场**（式1）| 有剧情能力的商单：前 1/3 纯剧情养注意力，灾难场景本身就是产品 demo |
| **非常规用法-逐品展示**（式13）| 创意/信息差：每个都是「哦原来还能这样」的惊喜，内容密度极高 |

> 〔示范〕选题「宝宝积食怎么判断」→ 选 **结论先行-逐条论证（式10）**：先抛「娃半夜哭别只当闹觉」，再逐条给信号，比平铺直叙更抓人。
> ⚠️ **商单走专属公式**：内容类型是商单时，结构层叠加 **「场-转-术-感-钩」五层**——场（真实生活场景代入）→ 转（从踩坑到「我终于找到」）→ 术（具体到能复述的选品理由/用法，不说「好用」）→ 感（口语化情绪：「真的救命」比参数更服人）→ 钩（预埋互动）。商单详细做法见 `商单脚本` 能力，这里只提醒结构别漏层。

### ② 找切入点（用方法底座的「切入点三维」，别用最显眼那个）

同一选题换切入点就换了一群人。三维各给 1 个，挑最适合我的，说为什么：
- **换时机**（比显眼需求早一步）/ **换人群**（找非显眼使用者：爸爸/爷奶/孕妈）/ **换场景**（找侧面场景：不是「在家用」是「自驾堵车」）。
- 〔示范〕选题「宝宝积食」，显眼切入是「积食怎么办」；换场景切入「那次娃半夜哭，我才知道是白天那顿撑的」——有画面、能代入。

### ③ 选开头（用「前5秒切入八公式」选最配的一个，列名 + 一句模板）

> 标题决定要不要点开，前 5 秒决定要不要看完。从这八公式里挑**最配你这条**的一个，套它的模板写开头那整段（不是一句钩子）：

| 公式 | 一句话模板 | 最适合 |
|---|---|---|
| **背书/经验型** | `[身份/经历]+我才发现+[意外结论]` | 经验干货、方法分享 |
| **悬念/好奇型** | `[反常动作/场景]+结果+[出乎意料]，留个疑问不答` | 好物种草、测评 |
| **反常识/颠覆型** | `[XX]不是[常规认知]，是[颠覆认知]` | 干货纠偏、信息差 |
| **信息差型** | `网上都说[常见方法]，但我发现[冷门方法]` | 省钱攻略、冷门技巧 |
| **情绪共鸣型** | `[强烈情绪词]+[具体场景]+紧迫感` | 生活vlog、吐槽 |
| **痛点爆破型** | `[具体痛苦]+那[怎么解决？]+[给方向]` | 选购指南、方法教学 |
| **对比反转型** | `[常规做法/结果] vs [我的做法/结果]` | 好物对比、复盘 |
| **利益点/清单型** | `如果你[限定条件]，[数字]+[利益承诺]` | 好物清单、月龄指南 |

> ❌ 别用「大家好我是 XX 今天分享」——前 3 秒就被划走。
> 〔示范〕积食那条配 **反常识型**：「娃半夜哭，很多妈妈第一反应是闹觉，其实可能是白天那顿积着了。」

### ④ 出大纲（开头/中间/结尾骨架，每段一句话意图，先不写词）

大纲里同时排进 **结尾钩子**（用「结尾三留存钩子」选 1-2 种，可叠加）：
- **预告留存型**：「下期分享 XX，关注不错过」——适合系列/合集（预告的得真发，别透支信任）。
- **互动引导型**：抛一个**具体**开放问题邀评论（「你家娃几个月？评论区告我帮你看」），别问「大家觉得呢」这种空问题。
- **截图干货型**：视频里出现可截图的清单/流程图/尺寸表，并明说「可以截这张图」——适合干货/攻略，截图≈收藏。

### ⑤ 停下来对方向（**这步不能省**）

把「结构 + 切入点 + 开头公式 + 大纲」给我，问一句"这个方向对吗？要不要调？确认了我就写脚本"。**等我回应，再进 Phase 2。**（方向错了，词写得再好也白费。）

## Phase 2 · 脚本编导（按确认的大纲，写能直接念的口播稿 + 拍摄备注）

每段写**可以直接念的话**，不是方向。守 SKILL.md 的种草五底线。**全程拿「活人感五要素」当尺子——这条片子里到底有没有「看见人」：**

| 要素 | 写脚本时盯什么 |
|---|---|
| **人** | 这条到底说给谁听？开头就要让那群人觉得「说的是我」 |
| **场景** | 有完整生活画面，不是零散概念（哪天、在哪、什么光景） |
| **事件** | 有具体发生的事 + 过程，不是只甩结论（从哪一步到哪一步） |
| **情绪** | 唤起了什么（治愈/解压/羡慕/成就/共鸣），别是流水账 |
| **人性** | 触到底层渴望（孩子被好好对待 / 我也能过上想要的生活），才会被转发 |
> 干货堆砌 ≠ 脚本。一条「有产品没有人」的精致空壳，数据一定不行——先确认这五层在。

### ① 开头前 3 秒（按 Phase 1 选定的切入公式落地成真句子）
- 直接进事，套上一段确认的公式模板，**不要**自我介绍开场。

### ② 中间（把素材讲成故事，一个场景一个点，带画面细节）
- ✅〔示范〕「那天娃把辅食碗扣地上，我蹲下去才发现勺子太凉」——有人、有场景、有事件。
- ❌「辅食要注意温度、工具、时机三个要点」——像说明书，没人看完。
- 跟着定好的结构走：清单类就「每个产品=一个独立小故事+一个真实用过才知道的细节」；教程类就「编号步骤+每步一个常见错误」；改造自白类就「质疑→我的做法→现在的具体变化」。
- 我没给的素材你补，**必须标**「（这是我举的例子，换成你真实的 XX）」。

### ③ 结尾（按 Phase 1 选的留存钩子落地）
- ✅ 互动型「你家娃几个月？评论区告我，我帮你看怎么加」；截图型「这张判断信号图可以截一下」。
- ❌ 干巴巴「关注点赞收藏」、空泛「大家觉得呢」、纯催购「赶紧下单」。

### ④ 拍摄备注（每段配一句怎么拍：画面/动作/景别）——短视频画面比口播重要
- 活人感五要素的「场景/事件」就靠画面落地：多给一句「这里拍娃的反应特写」「这里拍改造前后对比全景」。

## 正反例

**选结构：**
- ✅「这是判断类干货 → 用结论先行-逐条论证（式10），先抛反常识结论」——点名真实框架、说清适配
- ❌「这条算干货型，就按干货来写」——泛泛而谈，没落到具体结构，等于没选

**找切入点：**
- ✅「半夜哭才发现积食」——换场景、有画面、读者代入
- ❌「积食的 5 个原因」——最显眼、最多人做、刷过就忘

**写中间（看见人）：**
- ✅「那天娃把碗扣地上，我蹲下去才发现勺子凉了」——人+场景+事件都在
- ❌「辅食要注意温度、工具、时机」——干货堆砌，没看见人，谁都不觉得在说自己

## 禁用（叠加 SKILL.md 通用禁用）

- 别跳过 Phase 1 直接写脚本；别用我编的"痛点共鸣/干货教程"类目代替真实结构名。
- 别用别赛道的场景举例；别甩金句、别硬凑工整结构。
- 别把"有产品没有人"的精致空壳当成脚本——活人感五要素缺层就先补，不是改字能救。
- 框架只嵌「名+怎么用」就够，别整篇搬全量方法论（全量在付费加速器）。
